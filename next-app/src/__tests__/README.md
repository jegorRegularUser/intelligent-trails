# Тесты базы данных

Комплексные тесты для проверки всех операций с MongoDB и Server Actions.

## Запуск тестов

```bash
# Запустить все тесты
npm test

# Запустить тесты один раз (без watch mode)
npm test -- --run

# Запустить конкретный файл тестов
npm test src/__tests__/db/mongodb.test.ts

# Запустить с покрытием кода
npm test -- --coverage
```

## Структура тестов

### 1. MongoDB CRUD тесты (`src/__tests__/db/mongodb.test.ts`)

Тестирует базовые операции с MongoDB:

#### Users Collection
- **CREATE**: Вставка пользователей, проверка уникальности email
- **READ**: Поиск по email, по _id, получение всех пользователей
- **UPDATE**: Обновление имени, пароля, нескольких полей
- **DELETE**: Удаление по _id, по email, массовое удаление

#### Routes Collection
- **CREATE**: Вставка маршрутов с/без waypoints
- **READ**: Поиск по userId, фильтрация по категориям, сортировка, пагинация
- **UPDATE**: Обновление названия, статуса избранного, тегов
- **DELETE**: Удаление с проверкой userId, массовое удаление

#### Indexes
- Проверка создания уникальных и составных индексов

## Отладка проблем

Если тесты падают:

1. **Проверьте MongoDB Memory Server**
   ```bash
   # Увеличьте таймаут в beforeAll
   beforeAll(async () => {
     mongoServer = await MongoMemoryServer.create();
   }, 60000); // 60 секунд
   ```

2. **Проверьте моки**
   ```typescript
   // Убедитесь что моки вызываются
   expect(vi.mocked(auth)).toHaveBeenCalled();
   ```

3. **Проверьте очистку данных**
   ```typescript
   beforeEach(async () => {
     await db.collection('users').deleteMany({});
   });
   ```

## Добавление новых тестов

Для добавления новых тестов:

1. Создайте файл в `src/__tests__/`
2. Используйте MongoDB Memory Server для изоляции
3. Мокайте внешние зависимости (auth, getDb)
4. Очищайте данные в `beforeEach`
5. Тестируйте успешные и ошибочные сценарии
