3 Проектирование информационной системы

Для проектирования данной информационной системы был выбран язык объектно-ориентированного подхода UML.
Унифицированный язык моделирования UML (Unified Modeling Language) – это язык визуального моделирования, который разработан для спецификации, визуализации, проектирования и документирования компонентов программного обеспечения, бизнес-процессов и других систем. Данный язык одновременно является простым и мощным средством моделирования, который может быть эффективно использован для построения концептуальных, логических и графических моделей сложных систем самого различного целевого назначения.

3.4 Блок-схема алгоритма ранжирования мест

Блок-схемы используются для визуализации алгоритмов и логики работы программы. Они показывают последовательность операций и условные переходы.

На рисунке 3.4 представлена блок-схема алгоритма ранжирования мест по качеству, направлению и разнообразию.

[ПРОМПТ ДЛЯ ГЕНЕРАЦИИ ДИАГРАММЫ 3.4 - Flowchart for Place Ranking Algorithm]
Create a flowchart for the place ranking algorithm:

1. START
2. Input: places[], currentPosition, destinationPosition, maxResults=5
3. For each place in places:
   - Calculate qualityScore (0-100):
     * Base score = 50
     * If has rating: +0 to +30 based on rating/5
     * If has popularity: +0 to +20
     * If has size (parks): +0 to +20 based on log10(size)
4. Filter by direction cone:
   - For each place, check if in ±90° cone toward destination
   - If ≥3 places in cone: use only those
   - Else: use all places
5. For each filtered place:
   - Calculate distance to currentPosition
   - Apply distance penalty:
     * 0-500m: no penalty
     * 500-2000m: -10 to -30 points
     * >2000m: -30 to -50 points
   - Check direction: if in cone +15, else -20
   - finalScore = qualityScore - distancePenalty + directionBonus
6. Sort places by finalScore (descending)
7. Select diverse alternatives:
   - Take best place
   - While selected.length < maxResults:
     * For each remaining place:
       - Find min distance to already selected
       - diversityScore = qualityScore + (minDist/300m * 30)
     * Add place with highest diversityScore
8. Return selected places
9. END

3.5 Блок-схема алгоритма построения маршрута

На рисунке 3.5 представлена блок-схема алгоритма построения интеллектуального маршрута с точками интереса.

[ПРОМПТ ДЛЯ ГЕНЕРАЦИИ ДИАГРАММЫ 3.5 - Flowchart for Route Building Algorithm]
Create a flowchart for the smart route building algorithm:

1. START
2. Input: startAddress, endAddress, categories[], transportMode
3. Geocode startAddress -> startCoords
4. If geocoding failed: return ERROR "Address not found"
5. Geocode endAddress -> endCoords
6. If geocoding failed: return ERROR "Address not found"
7. Initialize: currentPosition = startCoords, builtSteps = []
8. For each category in categories:
   - radiuses = [200m, 800m, 2500m, 6000m]
   - foundPlaces = []
   - For each radius in radiuses:
     * Query OSM Overpass API with (currentPosition, category, radius)
     * If found ≥5 places: break
   - If foundPlaces.length < 1: skip category
   - Rank places using rankPlaces(foundPlaces, currentPosition, endCoords)
   - Select top 5 alternatives
   - Choose best alternative as waypoint
   - Add to builtSteps with alternatives
   - Update currentPosition = waypoint.coords
9. Query Yandex Maps API to build route:
   - Input: [startCoords, ...waypoints, endCoords], transportMode
   - Get: geometry, distances[], durations[]
10. Merge route data with builtSteps
11. Return builtSteps with metrics
12. END
