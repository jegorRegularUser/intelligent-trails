# Валидация расстояний маршрута

## Проблема

Пользователи могут попытаться построить маршрут между очень удалёнными точками (например, Москва → Варшава), что приведёт к:
- Некорректной работе поиска мест (OSM оптимизирован для городских масштабов)
- Маршрутам через пустые участки без достопримечательностей
- Долгому времени построения
- Неудовлетворительному результату

## Решение

Добавлена умная валидация расстояний с предупреждающим модальным окном для межгородских маршрутов.

## Пороговые значения

```typescript
ROUTE_DISTANCE_LIMITS = {
  MAX_REASONABLE_DISTANCE: 50000,      // 50 км - показываем предупреждение
  RECOMMENDED_MAX_DISTANCE: 30000,     // 30 км - рекомендуемый максимум
  TYPICAL_CITY_WALK: 15000,            // 15 км - типичная городская прогулка
}
```

### Почему 50 км?

- ✅ **Москва центр → МКАД**: ~20-25 км - нормально
- ✅ **Москва центр → Химки**: ~30 км - допустимо
- ✅ **Москва центр → Зеленоград**: ~37 км - на грани, но ОК
- ⚠️ **Москва → Тула**: ~180 км - межгород, показываем предупреждение
- ⚠️ **Москва → Варшава**: ~1200 км - точно не для прогулок!

## Логика работы

```
Пользователь нажимает "Построить маршрут"
    ↓
Проверяем расстояние между стартом и финишем (по прямой)
    ↓
Расстояние > 50 км?
    ├─ НЕТ → Строим маршрут сразу
    └─ ДА → Показываем предупреждающее модальное окно
              ↓
              Пользователь выбирает:
              ├─ "Изменить маршрут" → закрываем окно, возвращаемся к форме
              └─ "Всё равно построить" → строим маршрут без дополнительных проверок
```

## Компоненты

### 1. `RouteDistanceWarningModal.tsx`

Предупреждающее модальное окно с:
- Иконкой предупреждения (жёлтый треугольник)
- Информацией о расстоянии в км
- Списком причин, почему это может быть плохой идеей
- Двумя кнопками: "Изменить маршрут" и "Всё равно построить"

### 2. `routeValidation.ts`

Утилиты для валидации:
- `shouldShowDistanceWarning(distance)` - нужно ли показывать предупреждение
- `calculateDirectDistance(start, end)` - вычисляет расстояние по прямой
- `formatDistance(distance)` - форматирует расстояние для отображения

## Особенности реализации

### Проверка только при наличии конечной точки
```typescript
if (finalEndPoint) {
  const distance = calculateDirectDistance(startPoint, finalEndPoint);
  if (shouldShowDistanceWarning(distance)) {
    // Показываем предупреждение
  }
}
```

Если конечная точка не указана (только промежуточные точки), предупреждение не показывается.

### Расстояние по прямой, а не по маршруту
Используется формула гаверсинуса для вычисления расстояния "по прямой" (as the crow flies), а не реальное расстояние по дорогам. Это быстро и достаточно точно для валидации.

### Не блокируем, а предупреждаем
Пользователь всегда может построить маршрут, нажав "Всё равно построить". Мы не ограничиваем функциональность, а информируем о возможных проблемах.

## UI/UX

- **Цвет**: Жёлтый (amber) вместо красного - это предупреждение, а не ошибка
- **Иконка**: AlertTriangle - визуально отличается от ошибки (AlertCircle)
- **Тон**: Дружелюбный, не категоричный ("может быть не лучшей идеей" вместо "нельзя")
- **Выбор**: Пользователь сам решает, продолжать или нет

- ✅ Форматирование расстояний (метры, километры, округление)
- ✅ Проверка пороговых значений

Все тесты проходят успешно ✅

## Метрики

- **Порог предупреждения**: 50 км
- **Типичная городская прогулка**: 5-15 км
- **Рекомендуемый максимум**: 30 км
- **Москва → Варшава**: 1200 км (24x превышение порога!)

## Безопасность

- Валидация происходит на клиенте (быстро, без задержек)
- Не блокирует функциональность (пользователь может продолжить)
- Не отправляет дополнительные запросы на сервер
- Использует существующую функцию `getDistanceInMeters` (проверенная формула)
