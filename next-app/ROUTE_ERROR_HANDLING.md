# Обработка ошибок построения маршрута

## Обзор изменений

Вместо стандартного `alert()` при ошибках построения маршрута теперь используется красивое модальное окно с двумя опциями:
1. **Попробовать снова** - перезапускает построение маршрута с теми же параметрами
2. **Удалить точку** - автоматически удаляет проблемную точку и перестраивает маршрут без неё

## Новые компоненты

### 1. `Modal.tsx` (`src/components/ui/Modal.tsx`)
Универсальный компонент модального окна с:
- Backdrop с blur-эффектом
- Закрытие по Escape и клику вне окна
- Блокировка скролла body
- Анимации появления/исчезновения
- Подкомпоненты: `ModalHeader`, `ModalBody`, `ModalFooter`

### 2. `RouteErrorModal.tsx` (`src/components/features/RouteErrorModal.tsx`)
Специализированное модальное окно для ошибок построения маршрута:
- Отображает сообщение об ошибке
- Показывает название проблемной категории (если доступно)
- Две кнопки действий: "Попробовать снова" и "Удалить точку"
- Использует иконки из `lucide-react`
- Полностью интернационализировано (ru/en)

## Изменения в существующих компонентах

### `RouteBuilderSidebar.tsx`

**Добавлено состояние:**
```typescript
const [errorModal, setErrorModal] = useState<{
  isOpen: boolean;
  message: string;
  categoryName?: string;
  failedWaypointId?: string;
}>({
  isOpen: false,
  message: "",
});
```

**Новые функции:**

1. `extractCategoryFromError(errorMessage)` - извлекает название категории из текста ошибки
2. `handleRetry()` - закрывает модалку и перезапускает построение маршрута
3. `handleRemovePoint()` - удаляет проблемную точку и автоматически перестраивает маршрут

**Логика обработки ошибок:**
- При ошибке от `buildCompleteRoute()` открывается модальное окно вместо `alert()`
- Автоматически определяется проблемная категория из текста ошибки
- При удалении точки проверяется, это промежуточная точка или финиш
- После удаления автоматически запускается перестроение маршрута

## Типы ошибок

Модальное окно обрабатывает следующие типы ошибок:

1. **Категория не найдена** - `"Не найдено мест категории "{category}" поблизости"`
2. **Адрес не найден** - `"Не удалось найти адрес: {address}"`
3. **Общая ошибка** - `"Произошла ошибка при построении маршрута"`

## Поток работы

```
Пользователь нажимает "Построить маршрут"
    ↓
buildCompleteRoute() возвращает { success: false, error: "..." }
    ↓
Открывается RouteErrorModal с сообщением об ошибке
    ↓
Пользователь выбирает действие:
    ├─ "Попробовать снова" → handleRetry() → buildCompleteRoute()
    └─ "Удалить точку" → handleRemovePoint() → удаление точки → buildCompleteRoute()
```
